# 📘 我的个人知识库维护手册

这份文档用于指导如何维护、更新和美化我的个人博客（GitHub Pages）。
核心原则：**以 Typora 为中心，所见即所得，不折腾代码。**

---

## 1. 📂 推荐的文件目录结构

保持良好的文件结构，是长期维护的基础。建议采用以下结构：

```text
MyBlog/  (根目录)
├── .git/                # Git 版本控制文件夹 (自动生成，勿动)
├── assets/              # 图片资源库 (Typora 自动存图位置)
├── exported/            # [重点] 专门存放导出的 HTML 文件的文件夹
│   ├── index.html       # 网站首页 (由 0.首页.md 导出)
│   ├── assets/          # 导出时要把根目录的 assets 复制一份到这里
│   ├──笔记A.html
│   └──笔记B.html
├── 0.首页.md             # 网站入口源文件
├── 1.学习笔记/           # 按分类存放 Markdown 源文件
│   ├── 高斯过程.md
│   └── 线性代数.md
└── README.md            # 本手册
```

> **💡 为什么要单独搞一个 `exported` 文件夹？**
> 这样可以把“源文件(.md)”和“发布文件(.html)”分开。GitHub Pages 的设置里，你可以把 Source 设置为 `root`，然后平时只把 `exported` 文件夹里的内容上传；或者简单点，直接混在一起也行，只要你找得到文件。
>
> **最简模式（如果不分类）：** 所有 .md 和 .html 都在根目录混放，简单粗暴，适合初期。

## 2. 🚀 Git 日常管理流程

我们不需要敲复杂的命令，只需要掌握“三板斧”。

### 初始化 (仅需一次)
在博客文件夹内右键 -> "Open Git Bash here" (或终端)：
```bash
git init
git remote add origin git@github.com:你的用户名/你的仓库名.git
```

### 日常更新三部曲 (每次写完笔记后)

当你导出完新的 HTML 文件后，执行下面三条：

**第一步：添加到暂存区**
```bash
git add .
```
*(注意有个点，表示添加所有变化)*

**第二步：确认提交**
```bash
git commit -m "更新笔记：这里写你更新的内容"
```

**第三步：推送到 GitHub**
```bash
git push
```
*(如果第一次报错，可能需要 `git push -u origin main`)*

### ⚡ 懒人脚本 (Windows)
在根目录下新建一个 `deploy.bat` 文件，填入以下内容。以后双击它就能自动上传：
```bat
@echo off
echo 开始上传博客...
git add .
git commit -m "Auto update blog"
git push
echo 上传完成！
pause
```

## 3. 🎨 如何美化 index 页面 (首页)

既然是 Typora 直出，我们就在 Markdown 语法范围内做到最美。

### 技巧 A：使用 Emoji 和表格布局
不要只写枯燥的列表。利用 Markdown 表格做一个“九宫格”导航。

**源码示例：**
```markdown
# 🌟 欢迎来到我的知识库

这里记录了我关于机器学习、数学和编程的思考。

| 📅 近期更新 | 🧮 数学基础 | 🤖 深度学习 |
| :--- | :--- | :--- |
| [2026年度计划](./plans.html) | [线性代数复习](./math-linear.html) | [Transformer详解](./dl-transformer.html) |
| [本周复盘](./review.html) | [高斯过程](./math-gp.html) | [PyTorch 笔记](./dl-pytorch.html) |
```

### 技巧 B：添加封面图
在 `0.首页.md` 的最开头，直接拖入一张漂亮的宽屏图片（Unsplash 上找）。
Typora 导出时，这张图就是你的网站 Banner。

### 技巧 C：使用引用块做公告
```markdown
> 📢 **置顶公告**：最近正在攻克 Time Series Analysis，欢迎交流。
```

## 4. 📑 如何给笔记页面添加目录 (TOC)

有两种方式让你的长文笔记拥有清晰的导航。

### 方法一：页面内目录 (In-Page TOC)
在你的 Markdown 笔记最开头（标题下方），输入：

```markdown
[TOC]
```
按下回车。Typora 会自动生成一个可点击的目录树。导出 HTML 后，这个目录依然可以点击跳转。

### 方法二：侧边栏大纲 (Sidebar) —— **强烈推荐**
这是 Typora 导出 HTML 最强大的功能。

1. **导出时设置**：
   点击 `偏好设置` -> `导出` -> `HTML`。
   勾选 **[x] 保留大纲视图 (Export with Outline)** (不同版本翻译可能不同，通常在 Style 区域)。

2. **效果**：
   导出后的网页，**左下角**或者**右侧**会自动悬浮一个按钮。点击它，会滑出一个侧边栏目录，无论你滚动到哪里都能看到结构。这是最接近专业文档体验的设置。

## 5. 🔗 核心注意事项 (防坑必读)

1. **图片路径**：
   一定要在 Typora 设置里把图片设为**“复制到 ./assets 文件夹”**且**“优先使用相对路径”**。这是网页能显示图片的关键。

2. **链接后缀**：
   在 Markdown 里写链接时，如果你想跳到另一篇笔记，链接地址最好直接写 `.html` 后缀。
   * ❌ `[高斯过程](./高斯过程)`  (GitHub Pages 可能会认为是下载文件)
   * ❌ `[高斯过程](./高斯过程.md)` (浏览器会直接把源码显示出来，不渲染)
   * ✅ `[高斯过程](./高斯过程.html)` (正确跳转)

3. **文件名规范**：
   尽量不要在文件名里包含空格（用连字符 `-` 代替）。虽然中文文件名现在支持得很好，但在某些极端网络环境下，英文文件名（如 `math-gaussian.html`）永远是最稳的。
